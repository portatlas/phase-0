Git and GitHub: The Differences

Git and GitHub are not the same thing. Git is a local version control system that allows you to track and manage changes to your work. GitHub is a web-based version control system that allows you and other people (friends and strangers) to work collaboratively. Despite the differences in their use and functionality, a lot of magic can happen when Git and GitHub are used together.

Cloning from GitHub to your Local Machine

To reiterate, Git is a version control system that runs on your local environment,whereas GitHub tracks and manages files on the web. What if, you find source code on GitHub that you would like to contribute to or make changes to the code to match your needs. Fortunately (assuming the licenses permit), you can 'clone' the repository on GitHub to your local environment via the git clone command and the HTTPS URL. For example:

git clone https://github.com/user/repo

After that command runs, you have an exact copy of the source code from GitHub on your machine as well. You can do anything your hearts content with the source code on your machine.
Pushing from your Local Machine to GitHub

Now that you have added some functionality to the code and would like to share this back to the GitHub community. You can push your changes to GitHub via the command git push origin.

git push origin my-new-function

In this case, "origin" is pointing to your repo on GitHub. You might be asked to supply your login credentials to GitHub and this is one place where you see Git and GitHub interacting with each other quite clearly.
